Languages: 

###HTML / CSS
Ce sont les langues qui contrôle la disposition d'une page web. En tant que frontender, je les utilise tous les jours: le markup html pour la structure des pages, le css pour les rendre jolies.

###Javascript
C'est le code qui tourne dans le navigateur qui rend le tout interactif. J'utilise le javascript depuis un moment, parfois avec la librairie jQuery. Je suis actuellement en train de me plonger dans l'apprentissage d'un framework js à part entière, Vue.js et/ou React.

###PHP
J'utilise essentiellement le PHP pour le code côté serveur. Il me sert surtout pour le templating des pages avec WP ou Kirby, ainsi que d'autres tâches du serveur telles qu'envoyer des mails par exemple.

Cms: 

Les systèmes de gestion de contenu – (CMS en anglais) – sont des frameworks qui font exactement ce que leur nom indique: ils aident à gérer le contenu d'un site web. Il y a un grand nombre de CMS différents sur le web.

###Wordpress
WordPress est le CMS le plus utilisé sur le web. C'est un très bon outil que j'ai utilisé dans plusieurs projets. S'il est bien configuré, il permet aux clients qui ne connaissent rien au web de gérer leur propre site web. Il y a de nombreux plugins disponibles qui permettent de faire ce que l'on veut rapidement. Mais WP a également quelques défauts qui me poussent parfois à l'éviter: les sites peuvent être très lents, c'est souvent de l'overkill, et surtout la sécurité laisse à désirer. Il faut constamment faire des mise-à-jour du code/des plugins, faire des backups et mettre en place des systèmes de sécurité car les sites WP est tout le temps la cible préférée des hackers.

###Kirby
Kirby est mon CMS préféré actuellement. Il n'y a pas de base de données, tout est enregistré dans différents fichiers en utilisant la syntaxe Markup. Ça donne des sites web super rapides! J'essaie d'utiliser Kirby dans un maximum de mes nouveaux projets.

###Joomla
Avec mon travail à Marly, j'ai dû m'habituer à utiliser Joomla pour mettre à jour et éditer des sites web, pour la plupart créés il y a bien quelques années. Travailler avec Joomla est assez pénible, j'ai plus l'impression qu'il faut se battre pour gérer le contenu plus qu'autre chose. Je ne conseille pas Joomla dans de nouveaux projets. Beurk.

Workflow: 

Voici les outils que j'utilise régulièrement, qui rendent les sessions de codage plaisantes et efficientes.

###Git
Le version control est nécessaire pour l'élaboration du code en général, pas seulement pour le web. Le VCS le plus connu est Git, que j'utilise lors de tous les projets.

###jQuery
jQuery est une librairie javascript énormément utilisée encore aujourd'hui. Je l'utilise fréquemment, il permet de mettre en place des choses rapidement avec une API sympa.

###Gulp
Gulp est un build system que j'utilise pour effectuer des tâches automatiquement. Avec différents node modules, Gulp peut minifier, rassembler le css/js. Un autre module très intéressant est Browsersync, qui recharge les pages dans différents navigateurs ou appareils lorsque je sauvegarde un fichier en local.

###Sass
J'utilise Sass comme processeur css principal.

###Bootstrap et Susy
Je me sers de Bootstrap pour rendre les pages rapidement adaptives.  Vu que je n'exploite que du système de grille de Bootstrap, je me suis récemment intéressé à utiliser plutôt Susy.

Future: 

###Mes futurs intérets
Le web ne cesse jamais d'évoluer et de nouvelles technologies apparaîssent toutes les années/mois/semaines?... Il y a seulement quelques années, le responsive web design n'existait même pas. C'est pour ça que les développeurs web se doivent d'être un minimum à jour avec les nouvelles technologies ou frameworks. Je suis encore très motivé à en apprendre plus et continuer à acquérir des nouvelles compétences.

En ce moment, je me suis fixer certains objectifs: en apprendre plus sur les animations et les interactions sur le web. Je veux donc apprendre à utiliser le framework Greensock ainsi que Snapsvg pour des animations svg plus complexes. Je veux également continuer à évoluer en tant que développeur javascript, utiliser la nouvelle version ES6 et NodeJS. Je veux également maitriser bientôt un des framework js qui domine: Vue, React ou Angular.
